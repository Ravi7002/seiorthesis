Comparison of Huffman Encoding: Heap vs Naive (Varying Unique Characters)
=========================================================================================================
Each file has the same total size (~500 KB) but a different number of unique
characters. This isolates the O(n log n) vs O(n^2) tree-building cost.
A positive Diff means Naive is SLOWER than Heap.

Notes:
  - 'n' = number of unique characters (alphabet size).
  - Heap tree building: O(n log n)
  - Naive tree building: O(n^2)  [linear scan for minimum each step]
  - As n grows, the gap between Heap and Naive should widen.
  - All files are ~500 KB to keep encoding time (O(N)) roughly constant.
  - Run order alternates per file (even=Heap first, odd=Naive first) to
    cancel out OS page-cache bias from reading the same file twice.
